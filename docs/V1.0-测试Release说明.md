# Agent Memory for Hermes 1.0.0-rc.6 测试 Release 说明

## 1. 发布定位

`1.0.0-rc.6` 是 V1.0 的第六个接入测试候选版本，目标是让单用户在本机 Docker 环境中把多个 Hermes profile 接入同一证据驱动记忆空间，并验证长期运行、安全治理、备份恢复和星图人工介入链路。它在 rc.5 阶段 A 身份层之上完成阶段 B：Subject 是恒星、普通 canonical entity 是行星，事实分类成为观察镜片，episode/arc/Vault 成为非天体 overlay。

它不是无人值守公网服务，也不是凭据的唯一保管库。测试期间应保留原系统数据并定期执行数据库/Vault 双份备份。

## 2. 版本内容

- 镜像：`agent-memory-api:1.0.0-rc.6`、`agent-memory-worker:1.0.0-rc.6`、`agent-memory-migrate:1.0.0-rc.6`；
- 数据库迁移：`0001_evidence_core` 至 `0012_subject_identity`；
- Hermes 插件：`integrations/hermes/agent_memory`，由安全安装脚本管理；
- UI：星图、事实治理、状态参数、报告和 Vault；
- 默认端口：`127.0.0.1:7788`。
- 离线源码包：由 `scripts/package-release.sh` 生成，排除 Git、密钥、数据库、备份与缓存，并附镜像 ID 和 SHA-256 清单。

## 3. Release gate 证据

最终 release-check 必须产生以下全部 PASS：

1. Ruff、97 项单元测试和前端生产构建；
2. 14 项正式 API/worker 集成测试，强制运行于一次性测试 API/namespace；
3. 本机 Hermes 源码 4 项 Provider 测试，其中真实写入测试只允许自动化 namespace；
4. worker 过期租约恢复与停机积压恢复；
5. `pg_dump` 校验和、空库恢复、全层计数和 Vault 实际解密；
6. ARM64 版本化镜像构建、Compose health 和 API readiness；
7. 星图真实浏览器登录、图谱、状态参数、报告和 Vault 页面检查。

执行命令：

```bash
HERMES_AGENT_ROOT="$HOME/.hermes/hermes-agent" bash scripts/release-check.sh .env
```

最终实测镜像 ID、架构和时间记录在本文第 6 节。

## 4. 接入测试建议

先使用新的 Hermes 测试 profile，依次验证：普通对话写入、跨 profile 召回、工具结果证据、显式历史召回、用户更正、forgotten 唤醒、Vault grant/revoke、状态暂停/模拟/重置。至少运行一周后检查整理报告和阶段记忆休眠策略，再决定是否扩大真实数据范围。

遇到错误时保留 `docker compose logs`、失败的 correlation ID、版本号和操作时间；不要把 Vault 明文或 `.env` 放入问题报告。

## 5. 已知限制

- 仅单用户、本机 Hermes；无多租户权限模型；
- 实体识别与 episode/arc 合并采用保守确定性规则，不做开放式故事生成；
- 默认语义通道为本地确定性向量，模型增强默认关闭；历史导入在该状态下只保存证据，不生成整句事实；
- UI 适合本地治理，不支持公网反向代理、外部身份源或移动端专项适配；
- 星图关系在浏览器 Canvas 中渲染；自动化验收同时使用实体搜索入口验证详情和治理操作，避免把无障碍树无法直接定位 Canvas 星体误判为功能失败。
- 自动化测试空间与用户主空间逻辑隔离但共用本地 PostgreSQL；测试证据仍按只读原则保留，默认从用户星图隐藏，可通过噪声开关审计。
- `rc.6` 不包含关系社区、重叠成员、人工星系/布局治理和主/子星系缩放联动；这些能力已在 rc.6 之后的阶段 C 工作树与独立 `7798` 影子实例完成工程验证，但不能反向视为 rc.6 发布内容。
