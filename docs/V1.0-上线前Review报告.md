# Agent Memory V1.0 上线前 Review 报告

> Review 日期：2026-07-20。候选源码：`1.0.0-rc.7`。范围仅限本仓库和全新隔离发布环境，未修改生产 Hermes、正式 namespace、在线数据库或现有 Provider 配置。

## 1. 结论

代码与隔离发布 Gate 达到 **GO**。当前容器属于使用生产数据副本验证的测试栈，根目录 `.env` 缺失不再是新部署阻断项；正式路径改为创建全新生产预部署容器。正式 Hermes 接入仍为 **NO-GO**，直到预部署空栈、备份恢复和单 profile 授权完成。

`rc.7` 可以作为待签署候选继续打包和灰度准备，不能据此自动重启当前标准容器或修改生产 Hermes。

## 3. 安全与运行审查

- API/worker/model-worker 根文件系统只读，`cap_drop=ALL`，启用 `no-new-privileges`；
- API 仅发布到 `127.0.0.1`；core worker 仅连接 internal backend；model-worker 单独拥有 edge；
- release 环境模型默认关闭，不含模型 key，不允许发送外部数据；
- release env 与 Vault key 权限要求为 `0600`/`0400`，测试 UI 密码仅初始化时显示一次；
- API readiness 使用数据库就绪检查，不再以进程存活代替可服务；
- npm 生产/开发依赖审计均为 0 个已知漏洞；锁定的 Python 生产依赖审计无已知漏洞；
- 仓库密钥扫描未发现真实凭据，只有脱敏器测试使用的伪 key；
- 日志无阻断错误。PostgreSQL 初始化时的 local `trust` 提示来自官方镜像本地 socket 初始化流程；正式网络访问仍使用密码且数据库不发布宿主端口。
