# Agent Memory for Hermes — V1.0 运行与配置设计

## 1. Compose 边界

正式栈包括 `postgres`、一次性 `migrate`、`api` 与 `worker`。PostgreSQL、migrate、worker 和 API 共享无外部出口的 `backend` 网络；API 额外加入 `edge` 网络并仅发布宿主 `127.0.0.1` 端口。该双网络设计已在当前 Docker Desktop 验证：仅挂 internal 网络时 Docker 未生成实际宿主端口，加入 edge 后 `127.0.0.1:7788` 正常可达。

PostgreSQL 不发布端口。持久数据位于项目目录 `data/postgres`，删除项目目录前必须先执行备份；该目录不进入 Git 或 Docker build context。

## 2. 配置层级

配置来源按优先级为环境变量、项目 `.env`、代码默认值。`.env` 不进入版本控制，`.env.example` 只提供字段，不得承载真实密钥。

| 变量 | 含义 | 约束 |
| --- | --- | --- |
| `AGENT_MEMORY_VERSION` | 运行与镜像版本 | 当前 `1.0.0-rc.5`；必须与 `VERSION` 一致 |
| `AGENT_MEMORY_DB_PASSWORD` | PostgreSQL 密码 | 安装时随机生成，不复用外部密码 |
| `AGENT_MEMORY_SERVICE_TOKEN` | Hermes Provider → API Bearer token | 至少 16 字符，安装时随机生成，可轮换 |
| `AGENT_MEMORY_NAMESPACE` | 统一共享命名空间 | V1 单值，默认 `hermes:user-primary` |
| `AGENT_MEMORY_API_PORT` | 本地 API/UI 端口 | 仅绑定 `127.0.0.1` |
| `AGENT_MEMORY_WORKER_POLL_SECONDS` | 空闲轮询间隔 | 0–60 秒范围内配置 |
| `AGENT_MEMORY_WORKER_LEASE_SECONDS` | worker 任务租约 | 默认 180 秒，范围 5–3600 秒；模型 Worker 启用时必须至少覆盖 `timeout × (retries + 1) + 30 秒`，否则拒绝启动 |
| `AGENT_MEMORY_VAULT_ROOT_KEY_FILE` | Vault 根密钥 Docker secret 路径 | 默认 `/run/secrets/vault_root_key`，禁止使用普通 `.env` 明文值 |
| `AGENT_MEMORY_UI_PASSWORD_HASH` | 星图本地管理密码的 scrypt 哈希 | 初始化脚本生成；不得保存明文密码 |
| `AGENT_MEMORY_UI_SESSION_SECRET` | 星图会话签名密钥 | 初始化脚本随机生成；仅 API 使用 |
| `AGENT_MEMORY_REPORT_INTERVAL_DAYS` | 整理报告周期 | 默认 7 天，1–365 天 |
| `AGENT_MEMORY_CANDIDATE_RETENTION_DAYS` | 候选参与整理时长 | 默认 30 天 |
| `AGENT_MEMORY_CURRENT_STATE_DAYS` | 一般当前状态 TTL | 默认 7 天 |
| `AGENT_MEMORY_WEATHER_STATE_HOURS` | 天气类当前状态 TTL | 默认 24 小时 |
| `AGENT_MEMORY_CONTINUITY_DAYS` | 跨 profile 连续性 TTL | 默认 14 天 |
| `AGENT_MEMORY_STAGE_DORMANT_DAYS` | 阶段事实进入休眠阈值 | 默认 90 天 |
| `AGENT_MEMORY_STAGE_FORGET_DAYS` | 阶段事实进入忘记阈值 | 默认 365 天 |
| `AGENT_MEMORY_MODEL_ENABLED` | 是否启用模型增强 | 默认 `false`；仅 worker 接收 |
| `AGENT_MEMORY_MODEL_NAME` | LiteLLM 模型标识 | 启用时必填，可指向外部或本地模型 |
| `AGENT_MEMORY_MODEL_API_BASE` | 自定义模型 base URL | 仅允许 HTTP(S)，可为空使用提供商默认值 |
| `AGENT_MEMORY_MODEL_API_KEY` | 模型 API key | 仅 worker 接收；不得记录或进入普通记忆 |
| `AGENT_MEMORY_MODEL_TIMEOUT_SECONDS` | 单次模型超时 | 默认 30 秒，最大 300 秒 |
| `AGENT_MEMORY_MODEL_MAX_RETRIES` | LiteLLM 重试次数 | 默认 2，最大 10 |

模型、embedding、生命周期和报告频率均已接入对应运行路径；互动状态高级参数存于 `state.settings` 并通过星图管理，不在 `.env` 复制两套来源。

## 3. 启动与迁移

```bash
bash scripts/init-local.sh
docker compose config
docker compose up --build -d
curl http://127.0.0.1:7788/health/ready
```

`migrate` 必须成功退出后 API/worker 才启动。迁移使用 Alembic，DDL 在 PostgreSQL 事务中执行。API/worker 镜像以 UID 10001 的非 root 用户运行。

## 4. 开发验证

```bash

uv run pytest

AGENT_MEMORY_INTEGRATION=1 \
AGENT_MEMORY_SERVICE_TOKEN='<本地令牌>' \
```

集成测试只连接本地 API，不要求将 PostgreSQL 暴露给宿主。数据库级计数、备份和恢复由 Compose 内的受控测试命令验证。

## 6. Worker 恢复验证

```bash
bash scripts/verify-worker-recovery.sh .env
```

脚本创建一个租约已过期、`attempt_count=1` 的正式 `extract_facts` 任务。worker 必须以数据库租约取回，最终状态为 `done:2`。2026-07-13 正式栈验证结果为 PASS。

## 7. 备份与恢复演练

```bash
backup_dir="$(bash scripts/backup.sh .env)"
bash scripts/verify-restore.sh "$backup_dir" .env
```

备份脚本以 `umask 077` 创建 `pg_dump` 自定义格式归档，并保存 Compose、运行配置、依赖锁和版本，同时生成 SHA-256 清单。验证脚本先验证清单，再恢复至临时数据库，对比 evidence、facts、episodes、arcs、jobs、Vault、状态配置/快照和报告计数，实际解密 Vault 后删除临时库。

2026-07-14 恢复后最终演练通过，计数为 `69:73:0:4:214:9:9:9:1:1`（evidence、facts、episodes、arcs、jobs、Vault entries/grants、interaction snapshots/settings、reports），并在恢复库中使用 Docker secret 实际解密 9 条 Vault 记录。根密钥不得与数据库归档放在同一普通备份中。
