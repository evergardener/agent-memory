# Agent Memory for Hermes — V1.0 阶段 B 实施验证报告

## 1. 结论

阶段 B 已在 `1.0.0-rc.6` 完成并通过完整 release gate。星图的天体语义现在稳定为：本地用户和 Hermes profile 人格是恒星，普通 canonical entity 是行星；事实分类、episode、arc 和 Vault 不再伪装成星系或天体。

本阶段没有新增数据库迁移，也没有写入新的实体归属。`planetary-v2` 是从现有 Subject、entity、fact、episode、arc 和 Vault 引用生成的只读查询投影，因而可以回滚前端/投影代码而不改写原始证据和事实。

## 2. 已实施范围

### 2.1 图 API

`GET /api/v1/graph/subgraph` 返回以下独立集合：

- `nodes`：仅包含 `celestial_kind=star` 的 Subject 和 `celestial_kind=planet` 的 canonical entity；
- `edges`：主体关系和由共同支撑事实形成的行星—行星关系；关系保留 `fact_ids`、`support_count` 和强度；
- `facts`：非天体事实注释，保留实体、主体、来源、证据数、抽取版本、生命周期、活跃度和敏感性；
- `episodes` / `arcs`：非天体的临时星座与长期星流，可回到全部支撑事实；
- `vault_markers`：非天体的脱敏保护标记，仅含目标和引用 ID，不含密文或明文；
- `projection` / `facets`：投影版本、阶段 C 状态和可用镜片维度。

API 支持重复的 `profile`、`fact_type`、`lifecycle`、`activity`、`sensitivity` 参数及 `updated_after`。这些镜片只改变事实、关系和 overlay 的观察范围，不改变 Subject/entity 稳定 ID。

### 2.2 前端

- 移除长期、阶段、当前、情节、脉络和 Vault 固定星系；它们分别改为观察镜片、星座/星流和保护标记；
- 主体恒星固定在中心；行星按实体类型着色，在原位置约 5px 范围内缓慢漂移，悬停和选中时停止；
- 保留动/静开关、拖拽、缩放、搜索、详情和实体/事实治理；
- 选择事实、episode 或 arc 时高亮参与行星，并沿有事实支撑的关系线播放光点；
- episode/arc 详情按时间列出支撑事实，可逐条进入事实详情并继续追溯 evidence；
- Vault 镜片只显示脱敏保护条目及受保护行星数量，不显示可利用的敏感内容；
- Canvas 外提供可访问的行星与 overlay 列表，键盘 Enter 可打开同一详情。

## 4. 真实 staging 与浏览器验证

验证只读取本项目已经隔离导入的 Hermes staging 数据，没有修改生产 Hermes 配置、会话数据库或线上数据。

| 空间 | Subject | 行星 | 事实 | episode | arc | Vault 标记 |
| --- | ---: | ---: | ---: | ---: | ---: | ---: |
| `hermes:user-primary` 默认可见投影 | 2 | 0 | 438 | 1 | 16 | 23 |
| `hermes:import-staging` | 2 | 1 | 32 | 0 | 0 | 0 |

`hermes:import-staging` 在长期、当前、环境观察镜片下分别返回 7、6、12 条事实，镜片切换前后保持同一个 `outage` 行星 ID。浏览器验证还确认：默认没有固定语义星系；稀疏数据不会被 Cytoscape 自动放大成巨型行星；动静开关、键盘详情、星座/星流列表、trace 展开正常；主空间与 staging 控制台均为 0 error。

真实 staging 中现有 `outage` 是旧抽取策略从长篇 Alertmanager 提示形成的低可读实体。阶段 B 没有删除或改写这条历史数据；它说明下一阶段在社区计算前仍应运行实体质量门禁和人工抽检，不能用布局算法掩盖输入质量问题。

## 6. 明确未完成范围

以下能力仍属于阶段 C/D，不能因阶段 B 完成而宣称可用：

- 从行星关系生成可解释、可重叠且稳定的星系社区；
- 深空、桥接行星、主归属与次级归属；
- 星系人工创建、命名、固定、排除、撤销与布局持久化；
- 主宇宙连续放大进入子星系、子星系缩小返回主宇宙；
- 社区摘要和真实数据可读性的大规模验收。

下一步应先设计并验证阶段 C 的关系边质量与社区候选报告，再实现可治理社区。至少三个行星、两条独立关系链和一个非主体关系的防单体门槛必须保留，不能为了视觉丰满降低数据真实性要求。
