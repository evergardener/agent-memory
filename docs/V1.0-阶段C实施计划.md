# Agent Memory — 阶段 C 关系星系实施计划

> 基线日期：2026-07-19；前置状态：`GOLD_READY`；实施状态：C1–C5 工程实现与隔离验证已完成，正式空间启用等待用户验收和单独授权。

## 1. 目标与非目标

阶段 C 要把已有 `planetary-v2` 行星投影升级为可解释、可重建、可重叠且可人工治理的关系星系。
用户已接受的三个金标社区只用于评测和回归，不直接作为生产社区写入数据库。

本阶段不改变原始证据，不把模型建议写成事实，不修改生产 Hermes 数据，也不把长期/阶段/当前重新
变成空间分类。凭据明文、内部组件、来源 ID、句子和无语义共同出现继续禁止进入星图。

## 2. 推荐算法路线

V1 不直接绑定 Louvain/Leiden。先实现并比较三条只读路线：

1. **阈值连通分量基线**：用于发现输入过滤、权重或测试错误，不作为最终效果上限；
2. **确定性加权核心扩张**：以高质量语义边形成核心，再根据邻接权重分配 bridge 和次级成员；这是
   当前推荐默认方案，天然支持软归属、解释和稳定阈值；
3. **固定种子 Leiden 对照**：只作为效果对照；若引入原生依赖、结果漂移或无法解释的收益大于价值，
   V1 不纳入正式运行时。

关系权重至少综合：关系类型、独立 evidence 数、独立 session 数、来源可信度、生命周期状态、时间
衰减和人工固定/排除。共同出现本身只能产生待审核候选，不能直接成为稳定边。社区算法只读取当前
有效的行星—行星关系；Subject—行星边不参与社区形成。

评测不能只拟合三个正例，还必须加入以下负例与不变量：

- WebUI/Studio 别名合并后不得形成自环；Agent Bridge 不成为行星；
- PostgreSQL 与 Tailscale 共同出现在资产清单中不得生成路径边；
- 两行星关系、单行星对象和多实体列举不得被强制包装为星系；
- 同一行星可拥有主归属和次级归属，但全宇宙只有一个 canonical entity；
- 输入顺序变化结果不变，小幅新增证据不应导致无关星系整体重排；
- 每个成员、角色和名称必须能返回 relation/fact/evidence 解释链。

算法 Gate：三个金标社区成员关系全部恢复；所有负例为零违规；重复运行与输入乱序结果一致；增量
样本达到稳定阈值前不改变主归属；任何无法追溯的成员都使本轮评测失败。

## 3. 实施批次

### C1：只读关系图与算法评测

实施状态：**已完成**。评测见
[`V1.0-阶段C社区算法评测报告.md`](V1.0-阶段C社区算法评测报告.md)，算法决策见
[`ADR-002-阶段C社区算法结论.md`](ADR-002-阶段C社区算法结论.md)。

- 建立规范别名、类型化关系和传输路径的领域对象，不新增数据库写入；
- 实现上述三条算法的统一输入/输出契约、固定参数和版本号；
- 对金标、负例、重叠社区和增量扰动生成可复现差异报告；
- 形成算法选择 ADR，确认默认算法、阈值、稳定策略与依赖成本。

完成条件：算法 Gate 全部通过，并明确选中方案；否则停留在 C1，不进入迁移。

### C2：持久化与重建任务

实施状态：**已完成并验证**。`0013_relation_galaxies`、幂等社区重建、生命周期联动、过期租约恢复、人工治理保留和迁移升降级回归均已通过。

- 增加 `entity_aliases`、`entity_relations`、`relation_facts`；
- 增加 `galaxies`、`galaxy_memberships`、`galaxy_membership_evidence`、`layout_preferences`；
- 增加幂等 `rebuild_communities` worker 任务、算法版本和输入快照哈希；
- 自动投影可整批重建，人工命名、固定、排除和位置偏好不能被重建覆盖；
- 生命周期更正、脱离、忘记和清除只使受影响关系/社区失效并重算。

完成条件：迁移可升级/回滚；重复任务不产生重复记录；故障中断后可恢复；备份恢复包含全部人工治理
数据；原始 evidence 哈希保持不变。

### C3：API 与治理契约

实施状态：**已完成并验证**。关系星系、主/子图、治理、布局、撤销和审计 API 已通过隔离数据库与真实 HTTP 回归。

- 扩展 `GET /api/v1/graph/subgraph?view=universe|galaxy&galaxy_id=...`；
- 实现人工创建、命名、固定、隐藏、成员固定/排除和重建接口；
- 实现 layout 读取/保存、乐观并发控制、撤销与完整审计；
- 所有响应返回成员角色、主/次归属、关系语义、路径和 evidence 解释摘要。

完成条件：权限、namespace、生命周期和 Vault 隔离测试全部通过；并发更新不会静默覆盖用户操作。

### C4：星图 UI 联动

实施状态：**已完成并验证**。主宇宙/子星系、类型化关系、关系证据入口、约 5px 漂移、悬停静止、动静开关、布局持久化和 Canvas 外操作入口均已实现。

- 提供命名、固定、排除、撤销、合并/拆分及 Canvas 之外的键盘操作入口；

完成条件：数据计数、节点详情、治理操作和缩放状态一致；刷新后布局与人工规则稳定；不再出现演示数据
或不可读内部实体。
