## 5. 安全缺陷与修复

最终金丝雀检查发现旧 v3 脱敏器对“`sk-…` 前紧邻其他掩码字符”的字符串存在边界遗漏。该结果被视为 Gate 失败而不是误报：

1. 脱敏规则升级为 v4，新增裸 provider API key 识别和回归测试；
2. 从只读备份重新生成项目内脱敏副本，没有修改源备份；
3. 新副本额外遮蔽 57 处 provider key 形态；
4. 不原地更新旧 evidence，而是销毁并重建整个影子数据库，保持“证据写入后不可变”原则；
5. 重建后导入器报告 `potential_sensitive_findings=0`，数据库裸 `sk-…` 金丝雀为 0。
